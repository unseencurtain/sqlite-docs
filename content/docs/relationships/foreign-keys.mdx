# Foreign Keys

A foreign key is a column or group of columns that creates a relationship between two tables.

It ensures that values stored in one table correspond to existing rows in another table.

Foreign keys help maintain data integrity and prevent invalid relationships.

## Why Use Foreign Keys?

Without foreign keys, a table could contain references to rows that do not exist.

For example:

| order_id | customer_id |
| -------- | ----------- |
| 101 | 1 |
| 102 | 999 |

If customer `999` does not exist, the data is inconsistent.

A foreign key prevents this problem.

## Basic Syntax

```sql
CREATE TABLE customers (
    customer_id INTEGER PRIMARY KEY,
    name TEXT NOT NULL
);

CREATE TABLE orders (
    order_id INTEGER PRIMARY KEY,
    customer_id INTEGER,
    FOREIGN KEY (customer_id)
        REFERENCES customers(customer_id)
);
```

The foreign key tells the database that every `customer_id` in `orders` must exist in `customers`.

## Inserting Valid Data

```sql
INSERT INTO customers (customer_id, name)
VALUES (1, 'Alice');
```

```sql
INSERT INTO orders (order_id, customer_id)
VALUES (101, 1);
```

Because customer `1` exists, the insert succeeds.

## Inserting Invalid Data

```sql
INSERT INTO orders (order_id, customer_id)
VALUES (102, 999);
```

Because customer `999` does not exist, the database rejects the insert.

## Foreign Keys and Relationships

Foreign keys are used to build relationships between tables.

### One-to-Many

```sql
FOREIGN KEY (customer_id)
    REFERENCES customers(customer_id)
```

One customer can have many orders.

### One-to-One

```sql
user_id INTEGER UNIQUE,
FOREIGN KEY (user_id)
    REFERENCES users(user_id)
```

One user can have one profile.

## Composite Foreign Keys

A foreign key can reference multiple columns.

Parent table:

```sql
CREATE TABLE enrollments (
    student_id INTEGER,
    course_id INTEGER,
    PRIMARY KEY (student_id, course_id)
);
```

Child table:

```sql
CREATE TABLE grades (
    student_id INTEGER,
    course_id INTEGER,
    grade TEXT,
    FOREIGN KEY (student_id, course_id)
        REFERENCES enrollments(student_id, course_id)
);
```

Both columns must match an existing row in the parent table.

## Deleting Referenced Rows

A database may prevent deleting rows that are still referenced by another table.

For example:

```sql
DELETE FROM customers
WHERE customer_id = 1;
```

If orders still reference customer `1`, the operation may fail.

## ON DELETE CASCADE

To automatically delete related rows:

```sql
CREATE TABLE orders (
    order_id INTEGER PRIMARY KEY,
    customer_id INTEGER,
    FOREIGN KEY (customer_id)
        REFERENCES customers(customer_id)
        ON DELETE CASCADE
);
```

When a customer is deleted, all related orders are deleted automatically.

## Summary

A foreign key:

- Connects tables together
- Enforces referential integrity
- Prevents invalid references
- Forms the foundation of table relationships
- Can optionally define behavior when rows are updated or deleted

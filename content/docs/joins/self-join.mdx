---
title: "SELF JOIN"
---

A self join joins a table to itself.

It is useful when rows in a table are related to other rows in the same table.

## Syntax

```sql
SELECT columns
FROM table_name AS t1
JOIN table_name AS t2
ON t1.column = t2.column;
```

Table aliases are required so SQLite can distinguish between the two references to the same table.

## Example Table

### employees

| employee_id | employee_name | manager_id |
|------------|--------------|------------|
| 1 | Alice | NULL |
| 2 | Bob | 1 |
| 3 | Charlie | 1 |
| 4 | David | 2 |

## Example Query

```sql
SELECT
    e.employee_name AS employee,
    m.employee_name AS manager
FROM employees AS e
LEFT JOIN employees AS m
ON e.manager_id = m.employee_id;
```

## Result

| employee | manager |
|-----------|----------|
| Alice | NULL |
| Bob | Alice |
| Charlie | Alice |
| David | Bob |

The table is joined with itself to connect employees to their managers.

## Another Example

Find pairs of employees with the same manager:

```sql
SELECT
    e1.employee_name,
    e2.employee_name
FROM employees AS e1
JOIN employees AS e2
ON e1.manager_id = e2.manager_id
WHERE e1.employee_id < e2.employee_id;
```

## Common Uses

- Employee-manager relationships.
- Organizational hierarchies.
- Category hierarchies.
- Parent-child relationships.

## Summary

- A self join joins a table to itself.
- Table aliases are required.
- Uses normal join types such as `INNER JOIN` and `LEFT JOIN`.
- Useful for hierarchical data.

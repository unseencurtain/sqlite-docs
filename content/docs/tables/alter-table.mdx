---
title: "Altering Tables"
---


SQLite supports a limited set of table modifications through the `ALTER TABLE` statement.

## Rename a Table

```sql
ALTER TABLE users
RENAME TO customers;
```

## Rename a Column

```sql
ALTER TABLE users
RENAME COLUMN username TO full_name;
```

## Add a New Column

```sql
ALTER TABLE users
ADD COLUMN phone TEXT;
```

You may optionally include constraints:

```sql
ALTER TABLE users
ADD COLUMN status TEXT DEFAULT 'active';
```

## Supported Operations

SQLite directly supports:
- Renaming tables
- Renaming columns
- Adding columns
- Dropping columns (SQLite 3.35.0 and newer)

## Dropping a Column

```sql
ALTER TABLE users
DROP COLUMN phone;
```

## Limitations

SQLite does not directly support:

- Changing a column's data type
- Reordering columns
- Adding a primary key to an existing table
- Removing constraints

For these operations, the usual approach is:

1. Create a new table.
2. Copy data into it.
3. Drop the old table.
4. Rename the new table.

Example:

```sql
CREATE TABLE users_new (
    id INTEGER PRIMARY KEY,
    username TEXT NOT NULL
);

INSERT INTO users_new
SELECT id, username
FROM users;

DROP TABLE users;

ALTER TABLE users_new
RENAME TO users;
```

## Verify Changes

View the updated schema:

```sql
.schema users
```
